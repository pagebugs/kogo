## 3. "미루면 안 되는 것" vs "미뤄도 되는 것"

### 3.1 미루면 안 되는 것 (Phase 1 마무리 전 필수)

| # | 항목 | 이유 | 우선순위 |
|---|------|------|----------|
| 1 | **모바일 반응형 검증** | 사용자 접근성 기본 조건 | 🔴 Critical |
| 2 | **Paywall 섹션 정리** | UX 혼동 방지 | 🟡 High |
| 3 | **Production 환경 배포** | 실제 운영 테스트 | 🔴 Critical |

#### 상세: 모바일 반응형 검증

```
검증 항목:
□ 720px 이하에서 Hero Section 가독성
□ KPI Cards 수직 정렬
□ Modal 전체 화면 표시
□ Sticky CTA Bar 터치 영역
□ Map 확대/축소 터치 반응
```

#### 상세: Paywall 섹션 정리

`newresult.html`의 Premium Teaser 섹션 처리 방안:

```
Option A: Coming Soon 스타일로 변경
  - "카카오 로그인" 버튼 비활성화
  - "곧 서비스 예정" 문구 추가

Option B: 제거
  - Inquiry CTA로 통합
  - Paywall 섹션 전체 숨김

권장: Option A (Phase 2 예고 효과)
```

## 4. Phase 2 시작 조건

### 4.1 최소 조건 (Minimum Viable Trigger)

다음 조건 중 **하나 이상 충족** 시 Phase 2 시작:

```
┌────────────────────────────────────────────────────────────┐
│ Phase 2 시작 트리거                                        │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  □ Inquiry 누적 50건 이상                                  │
│    → 데이터 기반 패턴 분석 가능                            │
│                                                            │
│  □ 재방문 User 30% 이상                                    │
│    → 로그인/저장 기능 니즈 확인                            │
│                                                            │
│  □ Admin 처리 병목 발생                                    │
│    → 자동화 필요성 검증                                    │
│                                                            │
│  □ "결과 저장" 피드백 3건 이상                             │
│    → User 기대 수준 확인                                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 4.2 Phase 2 우선순위 후보

Phase 2에서 진행할 기능 우선순위:

| 순위 | 기능 | 트리거 | 예상 개발 규모 |
|------|------|--------|---------------|
| 1 | 결과 저장 + 로그인 | 재방문/저장 요청 | M (2-3주) |
| 2 | 자동 응답 이메일 | 볼륨 증가 | S (1주) |
| 3 | 마이페이지 | 로그인 도입 후 | M (2주) |
| 4 | 리마케팅 시스템 | 비활성 리드 증가 | L (4주+) |

## 5. Phase 1 마무리 체크리스트

### 5.1 기술적 체크리스트

```
Phase 1 Close Checklist - Technical
─────────────────────────────────────────────────────

□ Production 환경 배포 완료
  └── 서버 설정
  └── 도메인 연결
  └── SSL 인증서

□ API 연동 검증
  └── Runcomm API (IP 허용)
  └── Kakao Map API
  └── Daum Postcode API

□ 데이터베이스 운영 준비
  └── 백업 스케줄 설정
  └── 인덱스 최적화
  └── 연결 풀 설정

□ 에러 처리/모니터링
  └── 500 에러 핸들링
  └── API 타임아웃 처리
  └── 로그 수집
```

### 5.2 UX 체크리스트

```
Phase 1 Close Checklist - UX
─────────────────────────────────────────────────────

□ Desktop 1920px 검증
  └── Hero Section
  └── Dashboard KPIs
  └── Admin 테이블

□ Mobile 720px 검증
  └── 반응형 레이아웃
  └── 터치 영역
  └── 모달 스크롤

□ User Flow 검증
  └── Landing → 시뮬레이션 → Dashboard → Inquiry → 완료
  └── 각 단계 전환 자연스러움

□ Admin Flow 검증
  └── Dashboard → List → Detail → Status → Order
  └── 판단에 필요한 정보 가시성
```

### 5.3 비즈니스 준비 체크리스트

```
Phase 1 Close Checklist - Business
─────────────────────────────────────────────────────

□ 운영 프로세스 정의
  └── Inquiry 응대 SLA (예: 24시간 내 연락)
  └── Status 변경 기준
  └── Order 생성 권한

□ 성과 측정 지표 정의
  └── Inquiry 전환율
  └── Source별 리드 품질
  └── Admin 처리 시간

□ 피드백 수집 채널
  └── User 피드백 (어디서?)
  └── Admin 피드백 (어떻게?)
```

## 6. 최종 요약

### Phase 1 상태

> ✅ **"닫을 준비가 되어 있습니다"**

남은 작업:
1. 모바일 반응형 검증 (🔴 Critical)
2. Paywall 섹션 정리 (🟡 High)
3. Production 배포 (🔴 Critical)

### Phase 2 진입 조건

**Inquiry 50건 누적** 또는 **재방문 User 30%** 또는 **결과 저장 요청** 발생 시

### Phase 2 첫 번째 작업

**"결과 저장" 기능 + 카카오 로그인** (예상: 2-3주)
