# 터치애드 프로젝트 요건서

## 1. 프로젝트 개요

**프로젝트명**: 터치애드 가치제안 페이지
**상위 서비스**: Kogo SaaS Web (한국글로벌헬스케어협동조합)
**기간**: 2026년 1월
**담당**: 프론트엔드 개발

---

## 2. 서비스 개요

터치애드는 신용카드 이용정보와 교통카드 결제정보를 활용한 위치 및 맥락 기반 초개인화 타게팅 광고 플랫폼입니다.

- **데이터 규모**: 하루 50만 건 이상의 카드 결제 데이터
- **타겟**: 실제 구매력을 가진 323만 명의 소비자
- **특징**: 시간·장소·상황을 고려한 정밀 타겟팅, 높은 비용 효율성

---

## 3. 타겟 고객 (세일즈 타겟)

| 세그먼트 | 특징 |
|---------|------|
| 경쟁 심화 지역 내 의원 | 동일 상권 내 경쟁이 치열하여 신규 환자 유치에 어려움 |
| 비급여 진료 중심 의원 | 객단가가 높고 마케팅 효과를 크게 기대할 수 있음 |
| 마케팅 효율 개선 희망 의원 | 기존 마케팅 채널에서 명확한 효과를 보지 못함 |
| 데이터 기반 경영 도입 의원 | 감에 의존하는 마케팅에서 벗어나 데이터 분석을 통해 과학적 마케팅 원함 |
| 신규 개원의 | 개원 초기 병원 인지도를 높이고 안정적인 환자층을 빠르게 확보 필요 |

## 5. USP (Unique Selling Proposition)

### Headline
"원장님의 진료권 안에 있는 분들이 가장 필요한 순간에 원장님의 병원을 떠올리게 합니다."

### Sub-copy
대한민국 323만 명의 카드 결제와 교통 이용 동선을 실시간으로 분석하여, 우리 병원과 가까운 곳의 환자에게 최적의 의료 서비스를 제공하세요.

### 핵심 가치
1. **실구매력 기반**: 단순 검색이 아닌 실제 소비 행태를 기반으로 우리 병원에 꼭 필요한 분들만 선별
2. **실시간 동선 기반**: 병원 인근 지하철/버스 하차와 동시에 정보를 전달하여 내원 편의성 극대화
3. **경영 효율 극대화**: 클릭당 50원의 합리적인 비용으로 타 매체 대비 압도적인 환자 도달률 보장

## 6. 구성 요건 (가치제안 페이지)

### 6.1 Hero Section (CTA)

**메인 헤드라인**:
"원장님의 진료권 안에 있는 분들이 가장 먼저 떠올리는 주치의가 되세요."

**서브 메시지**:

**A. 환자의 생활 동선에 우리 병원을 놓이게 하세요**
- 의미: 환자가 병원이 필요한 순간 가장 먼저 떠올리는 '최초 상기(Top-of-Mind)' 병원이 되는 전략
- 팩트: 환자의 30.5%는 거주지나 직장에서 가까운 의료기관을 선택합니다. (병원 선택 기준: 거주지 또는 직장에서의 거리, 한국갤럽 2022)

**B. 건강에 투자 의지가 있는 323만명의 가망 환자를 만나세요**
- 의미: 검증된 323만 명의 소비 행태를 통해 '의료 서비스가 꼭 필요한 사람'을 찾는 기술
- 팩트: 카드 소비 행태와 라이프스타일 분석을 통해 건강 관리에 적극적인 고관여 유저 식별

**C. 시간, 장소, 상황에 맞춰 필요한 순간에 의료 정보를 제공하세요**
- 의미: 광고를 '공격적 세일즈'가 아닌 지역 사회의 '주치의로서 제공하는 적절한 배려'로 포지셔닝
- 팩트: 사용자의 시간, 장소, 상황에 맞춰 필요한 순간에 정보 제공

## 7. 업무 프로세스

### 1단계: 가치 제안
- **동선 분석**: 환자의 생활 반경을 데이터로 이해
- **고관여 유저**: 건강에 투자 의지가 높은 323만 명의 데이터 기반
- **비용 효율**: 경영 부담을 낮추는 합리적인 도달 비용
- **플랫폼 신뢰도 배지**: 대형 파트너사 로고 노출로 매체 신뢰도 확보
- **성과 증명**: "실시간 타겟팅 시 노출 대비 실적 90.7% 달성" 수치로 데이터의 실효성 증명
- **의료법 준수 강조**: 전문 검수를 거친 정보성 메시지 송출로 병원의 품격 유지

### 2단계: 주소 입력 및 즉시 데이터 도출 (추후 구현)
- 원장님이 진료권(Address)을 입력하면 플랫폼의 공신력을 숫자로 즉시 보여줌
- 입력창: [ 우리 병원 주소를 입력하여 잠재 환자 도달 범위를 확인하세요 ]

### 3단계: 컨택 정보 입력 (Lead Generation) (추후 구현)
- 성함, 병원명, 진료과목, 연락처 입력 유도
- 상세 경영 진단 및 지역 동선 분석을 위해 담당 주치의(매니저) 배정

### 4단계: 프리미엄 리포트 제시 (Heatmap 예시)
- 내부 데이터 가공을 통해 제공될 결과물 기대감 심어주기
- L-DMP 기반의 결제 패턴 히트맵(Heatmap) 예시 제시
- 금융/통신 데이터를 결합한 리마케팅 시나리오 제안

## 8. 기술 요건

### 8.1 기본 요건
1. 한글로 커뮤니케이션
2. touch 폴더 하위에 Kogo SaaS Web의 하위 서비스인 터치애드 서비스를 위한 HTML 프로토타입 구축
3. css 폴더 하위의 style.css token을 기본으로 사용
4. 1920 PC 버전을 기본으로 720 미디어쿼리 반응형 웹으로 설계

### 8.2 프로토타입 구현 요건
- 랜딩페이지 (index.html) 설계
- 터치애드는 Kogo 하위의 단위 서비스로 global GNB와 향후 연계예정이므로 별도의 Sub menu 구성 없이 팝업 레이어 형태로 프로세스 진행 (One page)
- 랜딩페이지는 가치제안을 전달

### 8.3 디자인 스타일
- BW 그레이스케일로 작업 (후반 작업 시 디자인 스타일 적용)
- SUIT 폰트 적용

### 8.4 레이아웃 시스템
- 그리드 기반 반응형 사이트 구축
